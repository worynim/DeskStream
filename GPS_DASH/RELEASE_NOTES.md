# 🚀 Release Notes - GPS OLED4 Dashboard
 
## [v2.3.0] - 2026-04-01
### 🔊 물리 버튼 햅틱(오디오) 피드백 추가
- **패시브 부저 연동**: 7번 핀에 연결된 부저를 통해 버튼 터치 시 즉각적인 청각적 피드백을 제공하도록 개선했습니다.
- **동작별 차별화된 알림음**: 
    - **Short Press**: 짧고 높은 음 (3000Hz, 50ms) 재생.
    - **Long Press**: 길고 묵직한 음 (2000Hz, 200ms) 재생.
- **명시적 펄스 제어**: `Smart_Info_Station`의 `beep()` 함수를 참조하여, `delay()`와 `noTone()`을 조합한 명확한 알림음을 발생시킵니다.

## [v2.2.0] - 2026-03-25
### 🛰️ GPS 신뢰성 및 데이터 수신 혁신
- **UBX 전용 메시지 제어**: u-blox NEO-6M 모듈의 설정을 NMEA($PMTK) 방식에서 UBX 바이너리(`CFG-MSG`, `CFG-RATE`) 방식으로 전면 전환하여 설정 성공률을 극대화했습니다.
- **가시 위성(GSV) 직접 파싱**: Adafruit GPS 라이브러리가 기본적으로 지원하지 않는 '가시 위성 수(In-View)' 정보를 GSV 문장에서 직접 추출하여 위성 상태 화면에 표시합니다.
- **지능형 시간 업데이트**: 위치 고정(Fix)이 완료되기 전이라도, 위성으로부터 유효한 시간 패킷이 수신되면 즉시 화면 시간을 갱신하도록 로직을 개선했습니다.
- **이중 초기화 충돌 해결**: `GPS.begin()`의 내부 라이브러리 동작이 하드웨어 시리얼 설정을 덮어쓰는 문제를 해결하여 GPS 데이터 수신 안정성을 확보했습니다.
- **실시간 데이터 모니터링**: 5초 이상 GPS 데이터가 들어오지 않을 경우 시리얼 및 웹 로그에 즉시 경고를 띄워 하드웨어 연결 상태를 진단할 수 있게 했습니다.

## [v2.1.0] - 2026-03-25
### 💾 설정 보존 및 편의 기능 강화
- **비휘발성 메모리(NVS) 저장**: `Preferences` 라이브러리를 도입하여 4개 OLED의 표시 모드 및 화면 반전(Flip) 상태를 재부팅 후에도 유지하도록 제작했습니다.
- **주행 데이터 리셋 기능**: 10번 핀(OLED 3번 버튼)을 길게 누르면 현재 누적된 주행 시간과 거리를 즉시 초기화하는 기능을 추가했습니다.
- **코드 안정성**: 부팅 시 저장된 설정을 로드하고 하드웨어 레지스터에 즉시 적용하는 파이프라인을 구축했습니다.

## [v2.0.0] - 2026-03-25
### 📺 사용자 정의 멀티 대시보드 엔진 (Major Update)
- **OLED 독립 렌더링**: 4개의 OLED가 각각 고정된 정보를 보여주던 방식에서 벗어나, 어떤 화면에서도 원하는 정보를 띄울 수 있는 **동적 매핑 엔진**을 도입했습니다.
- **8가지 디스플레이 모드**:
    1. **시계/날짜**: 시간과 연월일을 한눈에 확인.
    2. **속도계**: 아날로그 스타일의 게이지.
    3. **나침반**: 부드러운 회전 애니메이션의 방위계.
    4. **위성 상태**: 사용 위성/가시 위성수 및 DOP 정밀도.
    5. **위도/경도 (New)**: 현재 위치의 정밀 좌표 표시.
    6. **고도 (New)**: 볼륨감 있는 산 그래픽과 고도계.
    7. **달력 그리드 (New)**: 오늘 날짜가 반전 강조된 실제 달력 모양 뷰.
    8. **주행 통계 (New)**: 실시간 주행 시간(Timer) 및 누적 이동 거리(Trip).

### 🔘 4버튼 멀티 제어 시스템
- **독립 모드 전환**: 각 OLED 하단에 매핑된 4개의 버튼(GPIO 1, 4, 10, 9)을 통해 개별적으로 화면 콘텐츠를 순환(1~8번) 시킬 수 있습니다.
- **스마트 롱 프레스 (Instant Trigger)**: 버튼을 800ms 이상 누르면 손을 떼지 않아도 그 즉시 동작이 발동되는 고성능 버튼 엔진을 구현했습니다.
- **버튼 기능 매핑**:
    - **Short Press**: 해당 OLED의 디스플레이 모드 변경.
    - **Long Press (OLED 1)**: 화면 전체 반전(HUD 모드) 루핑.
    - **Long Press (OLED 4)**: 데모 시뮬레이션 모드 토글.
    - **Long Press (OLED 2,3)**: 기능 확장을 위한 예약(Reserved).

### 🎨 UI/UX 및 데이터 로직 고도화
- **그래픽 개선**: 고도계의 산 라인을 3픽셀 두께로 보강하여 시인성 확보.
    - **캘린더 엔진**: Sakamoto 알고리즘을 통한 요일 자동 판별 및 KST 자정 날짜 롤오버 로직 적용.
    - **주행 통계 엔진**: 정지 시 타이머 일시 정지와 속도 적분 기반의 트립미터 데이터 산출 로직 구현.
    - **고급 가독성**: `maniac`, `bpixeldouble` 등 세련된 폰트를 상황에 맞게 최적 배치.

## [v1.9.0] - 2026-03-25
### 🛡️ 보안 및 안정성 강화 (Critical Fix)
- **WiFi AP 보안 적용**: 무인증 접속을 차단하기 위해 AP 비밀번호를 적용했습니다.
- **OTA 프로세스 개선**: 펌웨어 업데이트 완료 후 HTTP 응답이 클라이언트에 정상 전달되도록 500ms 지연 후 재시작하도록 수정했습니다.
- **GPS 시간 버그 수정**: UTC 자정(00:00:00)에 GPS 시간을 미수신으로 오판하던 로직을 `GPS.fix` 기반으로 교정했습니다.

### ⚡ 메모리 및 아키텍처 최적화
- **String 제거 (Zero-Heap)**: 매 프레임(10Hz) 발생하는 힙 단편화 방지를 위해 핫 경로의 `String` 객체를 `char[]` 스택 버퍼로 전면 교체했습니다.
- **로그 시스템 개선**: 가변 `String` 기반 로그를 고정 크기 **링 버퍼(Ring Buffer)**로 교체하여 메모리 사용량을 예측 가능하게 만들고 `substring()` 오버헤드를 제거했습니다.
- **의존성 분리 (Callback)**: `extern` 함수 참조를 함수 포인터 콜백 패턴으로 전환하여 헤더 간 include 순서 제약을 해소했습니다.
- **코드 구조 분해**: 140줄의 거대 함수 `updateDashboard()`를 4개의 독립 렌더링 함수로 분리하여 유지보수성을 높였습니다.

### 🔘 인터랙티브 기능 추가
- **화면 반전 루핑 (GPIO 1)**: 버튼을 누를 때마다 [Normal -> 좌우반전 -> 180도 -> 상하반전] 순서로 하드웨어 레지스터를 실시간 제어하는 기능을 추가했습니다.
- **HUD 모드 지원**: 전면 유리에 투사하는 HUD 장착 시 필수적인 Mirror 기능을 소프트웨어 수정 없이 버튼만으로 설정할 수 있습니다.

## [v1.8.0] - 2026-03-15
### ⚡ Software I2C 고속 드라이버 적용
- **Direct Register Access 구현**: ESP32-C3의 GPIO 레지스터(`GPIO.out_w1ts`, `GPIO.out_w1tc`)에 직접 접근하는 고속 SW I2C 콜백 함수를 적용했습니다.

## [v1.0.0] - 2026-03-12
### 🗓️ 개요
- 기존의 WiFi 기반 정보 스테이션을 GPS 전용 대시보드로 리팩토링한 초기 버전입니다.
