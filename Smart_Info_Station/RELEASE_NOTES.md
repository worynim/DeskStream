# 🚀 Release Notes

## [v2.9.0] - 2026-04-13
### Changed
- **전 소스 파일 헤더 표준화**: 상점, 금융, 날씨 등 모든 데이터 소스 및 메인 펌웨어 소스 파일 상단에 작성자 정보(@worynim@gmail.com) 및 Doxygen 스타일 주석 블록을 도입하여 프로젝트 전문성을 강화했습니다.

## [v2.8.0] - 2026-04-05
### Added
- **도움말 페이지 설정 상태 표시**: 5번 페이지(버튼 설정 안내)에 현재 기기의 설정 상태를 시각적으로 추가했습니다.
  - **BTN 1 (Flip Mode)**: 현재 화면 회전 상태를 텍스트로 표시 (NORMAL, MIRROR H, 180, MIRROR V).
  - **BTN 2 (Brightness)**: 현재 밝기 단계를 4개의 사각형 바(Filled/Empty)로 시각화하여 표시.

### Fixed
- **부저음 길이 정밀 제어 (FreeRTOS Timer)**: 버튼을 누를 때 화면 갱신 작업(OLED 통신)으로 인해 부저음이 길어지던 현상을 해결했습니다. FreeRTOS 타이머를 사용하여 메인 루프의 부하와 상관없이 정확히 50ms 동안만 소리가 나도록 로직을 개선했습니다.

## [v2.7.3] - 2026-04-03
### Improved
- **주말 차단 로직(Market Hour) 정밀화**: 토요일 오전 US 장 마감 이후부터 일요일 전체 기간 동안 금융 데이터의 자동 업데이트를 중단하여 시스템 리소스를 절약합니다.
- **분 단위 장중 체크**: 개장(`08:30`) 및 폐장(`16:10`) 시간을 분 단위로 세밀하게 판단하도록 `isDomesticMarketOpen` 로직을 강화했습니다.
- **환율(USD/KRW) 주말 차단**: 증시뿐만 아니라 환율 정보도 주말에는 업데이트하지 않도록 차단 로직에 포함했습니다.

## [v2.7.2] - 2026-04-03
### Added
- **WiFi 공장 초기화 기능**: 기기 전원 인가 시 **BTN 1** 버튼을 누르고 있으면 저장된 WiFi 설정(SSID, Password)을 즉시 삭제하는 기능을 추가했습니다.
- **초기화 시각/청각 피드백**: WiFi 초기화 트리거 시 OLED에 "!!! WiFi RESET !!!" 메시지가 표시되며, 부저를 통한 경고음이 발생하여 상태를 직관적으로 알 수 있습니다.

## [v2.7.1] - 2026-04-03
### Improved
- **BTN4 롱 프레스 강제 업데이트 강화**: BTN4를 길게 눌러 데이터를 새로고침할 때, 시장 개폐 시간(장중 여부)에 상관없이 무조건 최신 데이터를 읽어오도록 개선했습니다. 0으로 표시되던 금융 데이터가 장외 시간에도 정상적으로 갱신됩니다.
- **전체 위젯 배경 수집 (Full Refresh)**: 강제 업데이트 시 현재 화면에 보이는 위젯뿐만 아니라, 활성화된 모든 위젯(1~3페이지 전체)을 배경에서 함께 갱신하여 페이지 전환 시 즉각적으로 최신 정보를 확인할 수 있도록 최적화했습니다.
- **데이터 수집 태스크 로직 정교화**: `dataTask` 내의 중복 업데이트 체크 로직을 `processed_mask` 기반으로 통합하여 효율성을 높이고, 특정 상황에서 발생할 수 있던 루프 범위 오류를 수정했습니다.

## [v2.7.0] - 2026-04-02
### Added
- **시작 멜로디 (Startup Melody)**: 전원 인가 후 OLED 초기화 완료 시 즉시 경쾌한 4음 상승 멜로디(도미솔도) 재생. 청각적 부팅 완료 피드백 제공.
- **초기 로딩 2단계 우선순위 로드**: 부팅 시 현재 페이지(1페이지) 위젯을 먼저 `is_updating` 플래그와 함께 수집하여 각 OLED에 `"Updating..."` 메시지를 표시한 후, 나머지 페이지 데이터를 백그라운드에서 조용히 수집. 기존에는 모든 데이터가 로드될 때까지 1페이지 OLED 2~4번이 공백으로 남는 문제 해결.
- **`MAX_DATA_PAGE` 매크로 도입** (`config.h`): 데이터 페이지 수(3)를 전용 상수로 분리. `SCREEN_MAP` 배열 크기, 각종 루프 범위, 웹 설정 UI 모두 이 상수를 참조하도록 통일하여 페이지 수 변경 시 단 한 곳만 수정하면 됨.

### Fixed (P0 - 즉시 수정)
- **`beep()` 블로킹 → Non-Blocking 전환**: 기존 `delay()` 기반 부저 구현을 `millis()` 예약 방식(`beep_end_ms`)으로 교체. `beep_tick()`을 `loop()` 내에서 매 사이클 호출하여 부저가 울리는 동안에도 웹 서버 응답·시계 갱신·버튼 폴링이 정상 동작.
- **마그네틱 부저 잔류 전류 차단**: `beep_tick()`과 `playStartupMelody()`에서 `noTone()` 직후 `digitalWrite(BUZZER_PIN, LOW)` 명시 적용. HIGH 상태 유지로 인한 발열·전력 낭비 방지.
- **버튼 `init()` 중복 호출 제거**: `setup()` 내에서 동일한 `pinMode(INPUT_PULLUP)` 설정이 두 번 실행되던 잔余 코드 삭제.
- **`W_FUTURES` 시장 시간 체크 누락 수정**: 국내 선물지수가 시장 개폐 시간 조건 검사에서 누락되어 있던 문제 수정. `isDomesticMarketOpen()` 조건에 `W_FUTURES` 추가.

### Fixed (P1 - 안정성)
- **`force_update` 레이스 컨디션 해소**: `btn4_long()`, `handleSet()`, `dataTask()` 세 곳에서 `force_update` 변수를 `portENTER_CRITICAL(&updateMux)` / `portEXIT_CRITICAL(&updateMux)`로 보호. 기존 `update_flag`와 동일한 Critical Section 패턴으로 통일.
- **`BUTTON_PIN` 미사용 잔여 코드 제거**: 단일 버튼 방식의 이전 버전에서 사용되던 `const int BUTTON_PIN = BTN4_PIN` 상수 삭제.
- **`showUpdateMessage` 이중 애니메이션 제거**: 함수 내부에 있던 `static int dots` 카운터가 모든 위젯 사이에서 공유되어 점 애니메이션이 뒤섞이는 문제 해결. 호출측(`loop()` 엔진)이 완성된 메시지를 전달하므로 함수 내부 카운터를 삭제.

### Refactored (P2 - 코드 품질)
- **하드코딩 `12` 전면 제거**: `SCREEN_MAP[12]` 배열, `for (i < 12)` 루프 등 5개 파일에 산재한 숫자 `12`를 모두 `MAX_DATA_PAGE * 4`로 교체. (`Smart_Info_Station.ino`, `data_manager.cpp`, `web_handlers.h`, `web_handlers.cpp`)
- **`handleRoot` HTML select 중복 코드 리팩토링**: 동일한 구조의 select 박스 생성 코드가 1~3페이지별로 3회 반복되던 것을 `pageTitles[]` 배열 + `MAX_DATA_PAGE` 기반 단일 루프로 통합. 약 45줄 → 15줄로 단축.
- **웹 설정 option loop 매직 넘버 제거**: `for (j <= 13)` 을 `for (j <= MAX_WIDGET_TYPE)`으로 교체.

## [v2.6.0] - 2026-03-28

### Added
- **스마트 자동 루핑 (Smart Auto-Loop)**: 데이터 수집 중일 때 페이지 전환을 일시 정지하는 로직을 추가하여, 수집 도중 페이지가 넘어가는 현상을 방지.
- **위젯별 독립 수집 타이머**: 글로벌 타이머 대신 각 위젯별 개별 수집 주기를 관리하여, 루핑 모드에서 모든 정보가 주기적으로 정확히 최신화되도록 보장.
- **폰트 중앙 관리 매크로**: `config.h`에 `MAIN_NUM_FONT` 매크로를 정의하여, 시스템 전역의 숫자 폰트를 한 번에 변경할 수 있는 편의성 제공.
- **즉각적인 수집 태스크 반응**: 페이지 전환 시 백그라운드 태스크를 즉시 깨워 가시성 있는 위젯들의 업데이트 상태를 최우선으로 체크하도록 개선.

### Improved / Fixed
- **화면 깜빡임(Flicker) 제거**: `redraw_current_page()`의 중복 버퍼 정제 로직을 삭제하고 개별 위젯 단위로 렌더링을 최적화하여 표시 전환 속도 대폭 개선.
- **초기 밝기 최적화**: OLED의 기본 밝기를 100%에서 **25%**로 하향 조정하여 번인 방지 및 눈부심 최소화.
- **웹 설정 UI 개편**: 드롭다운 메뉴의 서비스 순서(달력을 첫 번째로, 사용 안 함을 마지막으로) 및 안내 문구 페이지 순서 재정렬.
- **주석 최신화**: 핀 설정 및 페이지 제어 로직 등 코드 내 모든 한글 주석을 현재 배포 버전에 맞게 수정.

## [v2.5.0] - 2026-03-28
### Added
- **4버튼 독립 내비게이션 시스템 구축**: `GPS_DASH`의 검증된 버튼 구조체를 이식하여 4개 버튼의 Short/Long Press를 독립적으로 제어.
- **4버튼 독립 내비게이션 시스템 세분화**: 각 버튼의 짧게(Short)/길게(Long) 누름 기능을 분리하여 사용자 인터페이스 직관성 강화.
  - **BTN 1**: 짧게(Short) - 하드웨어 레지스터 기반 **화면 회전(Flip)** / 길게(Long) - (Reserved)
  - **BTN 2**: 짧게(Short) - **OLED 밝기(Contrast)** 4단계 조절 / 길게(Long) - (Reserved)
  - **BTN 3**: 짧게(Short) - (Reserved) / 길게(Long) - **자동 페이지 루핑(Auto Loop)** ON/OFF 토글
  - **BTN 4**: 짧게(Short) - **페이지 전환(Next Page)** / 길게(Long) - **데이터 강제 새로고침(Force Update)**
- **하드웨어 제어 기능 고도화**: 회전 설정 및 밝기 조절 시 즉각적인 레지스터 반영 및 비휘발성 메모리(Preferences) 연동.
- **버튼 기능 도움말 페이지**: 실시간 버튼 할당 정보를 확인할 수 있는 전용 **5번 페이지** 신설. IP 정보는 **4번 페이지**로 이동 배치.
- **웹 설정 대시보드 확장**: 자동 루핑 모드 ON/OFF 설정을 웹 UI 섹션(화면 배치 카드)으로 통합 연동.

### Improved
- **설정 영속성 강화**: 화면 회전, 밝기, 루핑 모드 설정을 `Preferences`에 저장하여 재부팅 시 자동 복구.
- **UI 레이아웃 최적화**: 웹 설정 화면에서 루핑 모드 설정을 화면 배치 카드 내부로 배치하여 사용성 개선.

### Fixed
- **변수 스코프 컴파일 에러 수정**: 업데이트 상태 관리 변수(`prev_updating_states`)의 선언 위치를 코드 상단으로 이동하여 빌드 안정성 확보.

## [v2.4.0] - 2026-03-28
### Refactored
- **프로젝트 전면 모듈화 및 아키텍처 개선 (5단계 완수)**: 비대해진 단일 `.ino` 파일의 의존성을 끊고 기능별 전용 C++ 모듈로 완전 분리.
  1. `display_utils`: OLED 그리기 및 고속 통신 헬퍼 함수
  2. `config.h`: 타입 선언(`WidgetType`) 및 글로벌 핀/상수 정의 공간
  3. `widgets_display`: 위젯별 화면 렌더링 처리 전담
  4. `data_manager`: 네트워크 모듈, 파싱, 그리고 FreeRTOS 백그라운드 코어 태스크
  5. `web_handlers`: 와이파이 설정 웹 포탈 인터페이스(`handleRoot`, `handleSet`)
- **이중 선언(ODR) 및 링커 에러 원천 차단**: C++ 표준에 맞추어 헤더(`.h`)에는 선언만, 소스(`.cpp`)에는 정의만 두어 모듈 간 충돌을 영구적으로 방지.

### Fixed
- **파싱 추출 로직 100% 원복 및 복구**: 파일 리팩토링 과정 중 유실되었던 기능들(네이버 금융 JSON API, 기상청 Stream 파싱, 유튜브 구독자 등)의 핵심 알맹이를 안전하게 재이식 완료.
- **컴파일 안정성 확보**: 괄호 누락으로 인한 펌웨어 빌드 오류 등 프로젝트 분할 과정상의 치명적 에러를 모두 깔끔히 수정.

## [v2.2.0] - 2026-03-12
### Fixed
- **NVS 플래시 수명 보호**: `handleSet()`에서 스크린맵 저장이 값 변경 여부와 무관하게 항상 실행되던 문제 수정. `changed` 블록 안으로 이동하여 불필요한 플래시 쓰기 방지.
- **멀티코어 레이스 컨디션 해결**: `update_flag`의 read-modify-write 연산에 `portMUX` Critical Section을 적용하여 `dataTask`(Core 0)와 `loop`(Core 1) 간 동기화 보장.
- **변수명 오타 수정**: `youtube_channal` → `youtube_channel`로 전체 프로젝트 내 일괄 변경.

### Refactored
- **`redraw_current_page()` 테이블 드리븐 전환**: 12개 위젯을 수동 if-else 체인으로 분기하던 50줄의 코드를 `widgets_info[]` 테이블 기반 14줄 루프로 리팩터링. 위젯 추가 시 한 곳만 수정하면 됨.
