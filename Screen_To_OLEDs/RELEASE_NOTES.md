# Release Notes - Screen_To_OLEDs

## [V1.3.0] - 2026-04-09

### ⚡ 시스템 안정화 및 FPS 병목 완전 해결 (Stuttering Zero)

간헐적인 끊김 현상과 FPS가 6~10까지 급락하던 고질적인 성능 문제를 해결하여, 30fps 이상의 부드러운 스트리밍을 구현했습니다.

#### 🔴 핵심 안정화: Python 스레드 경합 해결 (FPS 드롭의 주요인)
- **Tcl/Tkinter Mutex 블로킹 해결**: 
    - **원인**: 백그라운드 스레드에서 Tkinter 변수(`StringVar.get()` 등)를 직접 호출할 때, 메인 스레드가 UI 렌더링(Cocoa/AppKit)으로 바쁘면 Tcl 내부 뮤텍스 대기로 인해 프레임당 **50~200ms의 블로킹**이 발생했습니다. 이것이 6fps까지 떨어졌던 진짜 이유였습니다.
    - **해결**: 메인 스레드에서 50ms마다 설정값을 일반 Python 변수에 캐싱하고, 백그라운드 스레드는 이 캐시만 읽도록 구조를 변경하여 스레드 간 간섭을 완전히 제거했습니다.

#### 🔧 ESP32 펌웨어 최적화 (F1+F2 개선)
- **더블 버퍼링 도입**: 수신용 버퍼와 출력용 버퍼를 분리하여, 화면 전송 중에도 다음 프레임을 끊김 없이 수신할 수 있도록 개선했습니다.
- **수신 로직 비차단 방식 변경**: `pushCanvas()`(32ms 블로킹) 호출을 UDP 수신 루프 밖으로 이동하여, 전송 지연 시간 동안 쌓인 다음 프레임 패킷을 한꺼번에 처리할 수 있게 했습니다.

#### 🍏 macOS Retina 디스플레이 대응
- **자동 다운샘플링**: Retina 환경에서 물리 픽셀(2x)을 그대로 처리하던 오버헤드를 줄이기 위해, 캡처 직후 즉시 논리 해상도로 다운샘플링을 수행합니다. 이를 통해 이미지 처리 부하를 **4배 감소**시켰습니다.
- **상태바 정보 개선**: 실제 설정한 논리 해상도를 표시하고, Retina 감지 시 `(Retina2x)` 태그를 노출하도록 수정했습니다.

#### 📡 네트워크 송출 안정화
- **청크 간 미세 지연(Pacing)**: UDP 데이터 전송 시 청크 사이에 500μs의 딜레이를 추가하여, ESP32의 수신 버퍼 오버플로우로 인한 패킷 손실을 방지했습니다.

## [V1.2.0] - 2026-04-09

### 🚀 성능 최적화 및 1:N 스트리밍 기능 추가

디더링 연산 속도를 극대화하고, 여러 대의 보드에 동시 송출할 수 있는 기능을 추가하여 상용 수준의 퍼포먼스를 확보했습니다.

#### 🆕 추가된 기능
- **Numba JIT 고속 연산 엔진**:
    - **C 수준의 연산 속도**: 수치 연산 전용 컴파일러인 Numba(njit)를 도입하여, 복잡한 Stucki, Sierra 디더링 알고리즘도 FPS 저하 없이 실시간으로 구동됩니다.
    - **실시간 처리**: Python 루프의 한계를 극복하여 512x64 해상도에서 안정적인 고속 스트리밍을 실현했습니다.
- **Broadcast Mode (1:N 스트리밍)**:
    - **전체 송출 지원**: IP 주소 끝자리를 `.255`로 설정하여 동일 네트워크상의 모든 DeskStream 기기에 화면을 동시에 전송할 수 있습니다.
    - **기본 모드 설정**: 프로그램 시작 시 브로드캐스트 모드가 기본으로 활성화되어 별도 설정 없이 다중 기기 제어가 가능합니다.

#### 🔧 UI/UX 및 안정성 개선
- **오버레이 안전 여백(Safe Margin) 확보**:
    - **핸들 가시성 보정**: 캡처 영역이 모니터 끝에 닿아도 이동용 핸들이 가려지지 않도록 상단 40px, 좌우 10px의 안전 마진을 강제 적용했습니다.
    - **경계선 클램핑**: 오버레이 창이 모니터 화면 밖으로 나가는 현상을 방지하여 조작 안정성을 높였습니다.

#### 📋 사용 안내
- **설치**: 성능 최적화를 위해 `pip install numba`가 권장됩니다. (미설치 시 자동으로 일반 모드로 동작)
- **조작**: 
    - `Settings` 영역에서 `Direct`와 `Broadcast`를 자유롭게 전환하며 단일/다중 기기 송출을 제어하세요.

## [V1.1.0] - 2026-04-09

### 🚀 기능 확장 및 디더링 엔진 고도화

PC(macOS) 화면을 4개의 OLED 매트릭스로 구성된 DeskStream 하드웨어에 실시간 스트리밍하는 시스템의 영상 처리 엔진을 대폭 강화했습니다.

#### 🆕 추가된 기능
- **고성능 디더링 엔진 확장**:
    - **6가지 B&W 변환 모드 지원**: 기존 Threshold, Floyd-Steinberg 외에 **Atkinson, Stucki, Sierra, Ordered(Bayer)** 모드를 새롭게 추가했습니다.
    - **실시간 모드 전환**: 스트리밍을 중단하지 않고도 6가지 알고리즘을 실시간으로 변경하며 최적의 화질을 찾을 수 있습니다.
- **UI/UX 개선**:
    - **B&W 모드 라디오 버튼 그리드**: 늘어난 선택지를 한눈에 볼 수 있도록 2열 그리드 형태의 라디오 버튼으로 화면 설계를 변경했습니다.
    - **시각적 가독성**: 라디오 버튼의 폰트 크기를 조정하여 다른 UI 요소들과의 통일성을 확보했습니다.

#### 🔧 기존 기능 개선
- **디더링 최적화**: 모든 에러 확산형 디더링 모드 선택 시 Contrast 기본값을 2.0으로 자동 세팅하여 소형 OLED에서의 시인성을 극대화했습니다.
- **macOS 스트리머 고도화**:
    - **UI 개선**: 레이아웃(가로/세로) 설정을 라디오 버튼으로 변경하여 직관성 향상.
    - **캡처 위치 기억**: 스트리밍 중단 후 재시작 시에도 마지막 드래그 위치를 자동으로 복구.
    - **투명 윈도우 대응**: macOS 특유의 캡처 버그를 회피하는 4-선 독립 윈도우 오버레이 방식 적용.
- **ESP32 펌웨어 고도화**:
    - **인터랙티브 도움말**: BTN 2 롱프레스 시 나타나는 도움말 화면에 밝기 상태를 사각형 심볼(■■□□)로 시각화.
    - **지능형 화면 배치**: 도움말 화면 출력 시 하드웨어 버튼 위치와 일치시키기 위해 전체 플립(Flip) 설정을 일시 무시.

#### 📋 사용 안내
- **실행**: `python screen_streamer.py`
- **조작**: 
    - `B&W Conversion` 영역에서 실시간으로 디더링 알고리즘을 변경하며 화면 질감을 테스트해 보세요.
    - 하드웨어 BTN2를 짧게 눌러 밝기 조절, 길게 눌러 도움말 확인.
