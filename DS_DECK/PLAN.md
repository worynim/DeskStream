# DS DECK (DeskStream Deck) 프로젝트 계획서

## 1. 엘가토 스트림덱(Stream Deck) 동작 원리 분석
- **통신 구조:** 컴퓨터와 USB HID (일반적인 키보드/마우스 표준)로 통신합니다.
- **핵심 역할 분담:** 스트림덱 기기는 '입력 신호 전송'과 '수신된 화면 출력'만 담당하는 디스플레이가 달린 더미 키보드에 가깝습니다. 모든 동작(매크로 실행, 기능 작동, 화면용 아이콘 생성)은 PC의 백그라운드 클라이언트 프로그램이 직접 연산하여 기기로 전송하는 방식입니다.

## 2. DS DECK 아키텍처 및 통신/저장 방식 연구

DS DECK는 스트림덱처럼 PC 프로그램에 완전히 종속되지 않고, **독립적인 Bluetooth 키보드 및 디바이스**로 동작하는 것을 추천합니다.

### 2.1 하드웨어(보드) 동작
- **통신:** 스마트폰, 패드, PC 등 어디에나 연결할 수 있도록 **BLE HID 키보드** 방식을 메인으로 사용합니다. (WiFi/Web 서버 방식은 응답성 이슈와 네트워크 종속성이 생길 수 있어 무선 키보드로써의 장점이 반감됩니다.)
- **데이터 보존:** PC에서 설정한 이미지(128x64 1-bit)와 단축키 정보 등을 기기 내의 **비휘발성 메모리(LittleFS)**에 저장하여, 전원이 꺼진 후 재부팅해도 독립적으로 이전 설정과 아이콘을 띄울 수 있도록 설계합니다.

### 2.2 설정 어플리케이션(Client Tool) 아키텍처 (무선 통신 방식)
DS DECK 기기와 무선으로 **단축키 지정** 및 **이미지 전송**을 수행하기 위한 두 가지 무선 아키텍처 옵션입니다:

1. **ESP32 내장 Web Server 방식 (공유기 연결 Station 모드 확정)**
   - **동작 원리:** DS DECK 기기가 **가정 내 무선 공유기(WiFi)에 연결(Station 모드)**되어 백그라운드에서 웹 서버를 구동합니다. 사용자는 Mac/PC에서 `http://dsdeck.local/` 주소로 접속해 기기 내부에 심어진 설정 웹페이지를 띄웁니다.
   - **연결 편의성:** 설정 시 Mac의 공유기 연결을 끊을 필요가 없어 인터넷 단절의 불편함이 없습니다. (초기 1회만 공유기 비밀번호를 기기에 저장하는 과정을 거치면, 이후부터는 전원 인가 시 다이렉트로 공유기와 자동 연결됩니다)
   - **장점:** PC/Mac/Linux에 앱 설치가 전혀 필요 없으며, 브라우저 상에서 이미지를 올리면 빠른 속도로 무선(HTTP POST) 전송됩니다.
2. **BLE (Bluetooth) 무선 전송 방식 (Python 클라이언트)**
   - **동작 원리:** PC의 파이썬 앱 프로그램이 블루투스를 통해 DS DECK의 BLE 서버에 무선으로 접속합니다. 
   - **장점:** WiFi가 없는 환경에서도 구성 가능합니다.
   - **단점:** 이미지 데이터 전송 시 BLE 특성상 속도가 느려 1개의 버튼 이미지를 올리는 데 수 초가 소요될 수 있습니다. 또한, PC에 배포 앱을 설치/실행해야 합니다.

**결론 (무선 추천안):** 가장 세련된 방식이자 별도 앱 배포가 필요 없는 **1번 ESP32 웹 서버 기반 UI** 구조로 설계 변경을 제안합니다. 사용자의 브라우저 자체가 클라이언트 앱 역할을 완벽하게 수행합니다.

## 3. 버튼 4개 활용 방안 (Short / Long Press)
- **짧은 누름:** 설정된 BLE 키보드 단축키(Macro) 전송
- **길게 누름 (사용자 메뉴 기능):**
  - 특정 버튼 길게 누름: **프로필/페이지 전환** (예: 페이지 1, 2, 3으로 전환하여 총 12개의 단축키 프리셋 활용 가능하게)
  - 다른 버튼 길게 누름: **디스플레이 밝기 조절 모드** (Stats Monitor나 GPS_DASH의 코드 재활용 가능)
  - 다른 버튼 길게 누름: **화면 플립 (180도 회전)** (마찬가지로 코드 재활용 가능)

## 4. 단계별 구현 계획 (Step-by-Step)

시행착오를 없애고 체계적으로 개발하기 위해 다음 플로우로 분할 작업을 제안합니다.

* **Phase 1: 핵심 하드웨어 및 로직 검증 [적용완료]**
  - 4개의 버튼(짧게/길게 인지) 및 4개의 OLED 디스플레이 동시 렌더링 검증.
  - ESP32 BLE Keyboard 라이브러리를 사용해 하드코딩된 특정 키 입력이 PC/Mac으로 잘 전송되는지 테스트.
* **Phase 2: 이미지 및 데이터 구조 설계 (NVM 저장소) [적용완료]**
  - 이미지를 담을 바이너리 파일이나 배열을 프래시 메모리(LittleFS)에 저장하고 로드하는 기능 확인.
  - 단축키 조합(Modifiers, Keys)을 저장할 JSON 혹은 바이너리 구조 설계.
* **Phase 3: 무선 설정망(WiFi Web Server) 및 웹 UI 클라이언트 개발 [적용완료]**
  - ESP32에 AsyncWebServer를 구현하고 내부 플래시에 HTML/JS 파일을 호스팅.
  - 브라우저용 웹 프론트엔드 제작: 4개의 가상 버튼 UI, 파일 업로드 시 HTML5 Canvas를 이용한 이미지 리사이즈 및 흑백 디더링 처리 스크립트 작성.
  - 무선(POST Request)을 통해 브라우저에서 보드로 이미지 데이터 및 JSON 설정값 전송 확인.
* **Phase 4: Client(웹)와 하드웨어의 무선 연동 및 렌더링 [적용완료]**
  - 웹브라우저에서 단축키를 맵핑하고 이미지를 웹업로드하면 보드에서 WiFi로 수신, Flash에 저장.
  - 저장된 이미지가 메인 루프를 통해 OLED에 즉시 렌더링되도록 연결. 
  - 재부팅 시 해당 값을 완벽히 복원하는 무선 동기화 모듈 완성.
* **Phase 5: 고급 기능 및 완성**
  - 페이지 전환 기능 도입 (메모리에 여러 페이지를 저장하여 길게 눌렀을 때 뷰 스위칭).
  - 밝기 제어 및 자동 화면 플립 메뉴 구현.
  - 버그 픽스 및 전체 시스템 안정화.
