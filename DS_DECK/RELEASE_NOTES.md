# Release Notes - DS DECK

## [v1.3.0] - 2026-04-08
### Added
- **16진수(`0xXX`) HID 매핑 표준화**: 웹 대시보드의 모든 특수키 설정을 16진수로 표기하여 전문적인 설정 환경 구축 및 가용성 향상.
- **동작 피드백 강화**: 설정 저장 외 아이콘 삭제 등 주요 이벤트 발생 시 개별 부저음을 추가하여 사용자 경험 개선.

### Changed
- **매핑 체계 전면 개편**: 기존 수학적 오프셋 방식에서 **1:1 명시적 매핑(Explicit Mapping)**으로 전환하여 F-키 및 방향키의 오동작 가능성 원천 차단.
- **지능형 설정 파싱**: 펌웨어에서 `strtol`을 도입하여 구형 설정값(10진수)과 신규 설정값(16진수)을 모두 인식하는 하위 호환성 확보.

### Fixed
- **키 충돌 버그 해결**: 방향키 입력 시 ESC, Enter, Backspace 등 엉뚱한 키가 입력되던 매핑 테이블 충돌 완벽 수정.
- **F-키 전수 교정**: F1~F12 및 F13~F24 키가 라이브러리 공식 상수와 정확히 일치하도록 정밀 보정.
- **한글 타이핑 유실 방지**: 단축키 모드에서 언어 전환(F19/F20) 전 `KEY_RIGHT`를 자동 전송하여 한글 조합을 강제 확정(Commit)하는 로직 고도화.
- **UI 렌더링 수정**: 웹 대시보드에서 F3 키의 선택 상태(`selected`) 표시 오류 및 옵션 중복 노출 수정.

## [v1.2.0] - 2026-04-08
### Added
- **Karabiner-Elements 연동 한/영 강체 전환**: 단순 토글 방식에서 벗어나 F19(영어), F20(한글) 키를 이용한 절대적 언어 전환 기능 추가.
- **카라비너 설정 파일 제공**: `macro_language.json` 파일을 통한 간편한 macOS 입력 소스 제어 가이드 추가.
- **단축키 모드 확장**: 설정 대시보드(특수키 메뉴)에서 F18, F19, F20 키를 직접 선택하여 테스트 및 할당 가능.
- **L/R Modifier 구분**: CTRL, SHIFT, ALT, CMD(GUI) 키의 왼쪽/오른쪽을 개별적으로 지정할 수 있도록 UI 및 로직 개선.

### Changed
- **웹 대시보드 로직 고도화**: 매크로 저장 시 `[#ENG#]`, `[#KOR#]` 태그를 자동으로 식별하여 삽입함으로써 타이핑 시작 시 언어 상태를 강제로 일치시킴.
- **브라우저 모드(Spotlight) 안정성 향상**: 
  - 주소 입력 전 **F19(영어 모드)**를 강제 수행하여 영문 URL 입력 보장.
  - Spotlight 검색 결과 파싱 대기 시간을 **2.5초**로 상향 조정하여 정확한 접속 보장.
- **README 업데이트**: 카라비너 설치 방법 및 고도화된 언어 전환 설정 섹션 최적화.

### Fixed
- **F19/F20 키코드 수정**: 표준 HID Usage ID(0x6E, 0x6F)를 사용하도록 보정.


## [v1.1.2] - 2026-04-07
### 🌐 브라우저 런처 및 Spotlight 자동화 구현
- **브라우저 모드(Browser Mode) 신규 구현**: 기기 버튼 클릭 한 번으로 macOS의 Spotlight를 소환하여, 사용자의 **기본 브라우저**로 웹사이트 URL을 직접 실행하는 유니버설 런처 기능을 추가했습니다.
- **Spotlight 자동화 안정성 고도화**:
  - 명령어 입력 시 `KEY_RETURN` 물리 키 코드를 강제 배정하여 `Cmd+Enter` 미작동 버그 완벽 해결.
  - Spotlight 실행 직후 `KEY_BACKSPACE`를 1회 전송하여 기존 검색 잔여 검색어를 깨끗이 삭제하는 무결성 로직 추가.
  - URL 입력 시 기기 내부에 저장된 한영 전환 찌꺼기(`[#CAPS#]`)를 펌웨어 단에서 완벽히 배제하고 순수 문자열만 전송함으로써 타이핑 중 한/영 꼬임 현상을 하드웨어적으로 원천 차단.

## [v1.1.1] - 2026-04-07
### Fixed & Improved (코드 리뷰 기반 안정성 강화)
- **치명적 버그 수정**: BLE 연결 시 멀티미디어(Media) 및 단축키(Combo) 모드가 작동하지 않던 로직 오류를 수정했습니다.
- **보안 강화 (API Validation)**: 웹 서버의 모든 엔드포인트에 인덱스 범위(0~3) 검증 로직을 추가하여 메모리 예외 상황을 원천 차단했습니다.
- **데이터 무결성 확보**: JSON 설정 로드 시 파싱 에러 처리를 추가하고, 예외 상황에서의 파일 핸들 누수(Memory Leak)를 방지하도록 로직을 개선했습니다.
- **파일 시스템 안정화**: 아이콘 업로드 핸들러의 전역 변수를 정적 지역 변수로 변경하여 동시 업로드 시의 충돌 위험을 제거했습니다.
- **컴파일 호환성**: `#ifndef` 가드를 도입하여 `BUZZER_PIN` 중복 정의로 인한 빌드 에러를 해결했습니다.

- 웹페이지에서 특수키 설정 시 값이 저장되지 않거나 표시되지 않던 로직 오류 수정

## [v1.0.0] - 2026-04-07
### 🚀 주요 개선 사항 (한글 매크로 최적화 및 안정성 강화)

#### 1. 한글 매크로 고도화 (Korean Macro Optimization)
- **한글 오토마타 교정**: "안녕" -> `dkssPd`와 같이 일부 모음이 잘못 변환되던 매핑 테이블(jungList) 오류를 완벽히 수정했습니다. 이제 "안녕하세요"가 영타 `dkssudgktpdy`로 정확히 변환됩니다.
- **원본 보존 구조(korVal) 도입**: 웹 대시보드 새로고침 시 한글이 자모 분해되어 보이던 문제를 해결하기 위해 기기에 원본 한글 문자열을 별도로 저장하는 로직을 추가했습니다.
- **전송 속도 최적화**: 맥(Mac) OS 입력기에서 오타가 발생하지 않는 최상의 속도인 **5ms (Hyper Speed)**로 타이핑 딜레이를 재조정했습니다.

#### 2. 웹 대시보드(Web UI) 버그 수정 및 개선
- **요소 중복 생성 해결**: '기기에 적용' 버튼 클릭 시 버튼 설정 패널이 하단으로 무한 증식하던 치명적인 UI 버그를 수정했습니다.
- **데이터 용량 확장**: 전송할 문자열 길이를 기존 64바이트에서 **256바이트**로 대폭 늘려 장문의 매크로 저장이 가능해졌습니다.
- **JSON 처리 안정화**: 대용량 설정값 처리를 위해 `DynamicJsonDocument` 구조로 변경하여 메모리 부족 현상을 방지했습니다.

#### 3. 하드웨어 안정성
- **BLE 컴파일 에러 해결**: `setDelay` -> `setTapDelay` 함수명 오기를 바로잡아 최신 라이브러리 환경에서의 빌드 문제를 해결했습니다.
