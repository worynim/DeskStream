## 🚩 대원칙 (Core Rules)
1. **무결성 보존**: 모든 리팩토링은 현재 작동하는 기능을 1:1로 보존하는 것을 전제로 합니다.
2. **단계별 검증 (Checkpoint)**: 한 단계가 끝날 때마다 반드시 컴파일 및 실제 기기 업로드 테스트를 수행하며, 사용자 승인 없이 다음 단계로 넘어가지 않습니다.
3. **안전한 수정**: 대규모 덮어쓰기(`write_to_file`)를 지양하고, `replace_file_content`를 사용하여 기존 코드의 맥락을 유지합니다.
4. **Git 제어 준수**: 파일 수정 후에는 반드시 권한이 소멸됨을 인지하고, 사용자의 명시적인 "커밋해" 지시가 있을 때만 Git 작업을 수행합니다.

## 📅 단계별 세부 계획 (Roadmap)

### 1단계: 하드웨어 레이어 분리 (I2C_Platform)
- **목표**: `DisplayManager` 내의 저수준 I2C 코드를 추출하여 하드웨어 의존성 제거.
- **수정 범위**:
    - `[NEW] Hangeul_Clock/i2c_platform.h, .cpp` 생성.
    - `DisplayManager` 내의 `u8x8_byte_...` 콜백 및 `i2c_hw_task` 로직 이관.
- **검증**: 컴파일 성공 및 4개의 화면이 이전과 동일하게 출력되는지 확인.

### 2단계: 설정 관리자 모듈화 (Config_Manager)
- **목표**: 흩어진 설정 변수를 구조체화하고, 플래시 메모리 마모 방지 로직 도입.
- **수정 범위**:
    - `[NEW] Hangeul_Clock/config_manager.h, .cpp` 생성.
    - `SystemSettings` 구조체 정의.
    - 지연 저장(Lazy Save, 3초 대기) 메커니즘 구현.
- **검증**: 버튼으로 설정을 바꾼 후 재부팅 시 값이 유지되는지, 불필요한 직후 저장이 없는지 확인.

### 3단계: 렌더링 엔진 및 캔버스 분리 (Display Engine)
- **목표**: 그래픽 연산과 전송 엔진의 결합도 완화.
- **수정 범위**:
    - `DisplayManager` 내의 애니메이션 루프(`updateAll`)를 별도 모듈 또는 정제된 메서드로 분리.
    - 캔버스 객체 추상화 기반 마련.
- **검증**: 5가지 애니메이션(Scroll, Fade, Zoom 등)의 시각적 품질 변화가 없는지 대조.

### 4단계: 한글 시간 표기 정합성 고도화 ("열세시")
- **목표**: 24시간 표기법 및 한국어 고유어 수사 체계의 일관성 확보.
- **수정 범위**:
    - `HangeulTimeConverter` 클래스 내의 하드코딩된 로직을 설정 기반으로 정문화.
    - "열세시", "스무시" 등 특정 시간대 표기 규칙 검증.
- **검증**: 13시, 20시 등 경계 시간대의 텍스트 출력 결과물 확인.

### 5단계: 시스템 서비스 유닛화 (Input & Logger)
- **목표**: 버튼 입력 및 시스템 로그 기능을 독립 엔티티로 분리.
- **수정 범위**:
    - `Logger` 클래스 분리.
    - `InputHandler` 클래스를 통한 버튼 인터럽트 로직 정리.
    - `Hangeul_Clock.ino`의 핵심 비즈니스 로직 집중도 향상.
- **검증**: 전체 시스템 통합 테스트 및 웹 제어 기능 동작 확인.

## ⚠️ 리스크 관리 계획
- **파일 유실 방지**: `view_file`을 통해 항상 원본을 먼저 확인하며, 100라인 이상의 파일 수정 시 라인 수 대조를 수행합니다.
- **기능 퇴보(Regression) 방지**: 리팩토링 중 "더 나은 코드"라는 자만으로 기존의 복잡하지만 작동하는 로직(예: 인터럽트 구조)을 단순화하지 않습니다.
- **실패 시 복구**: 각 단계 시작 전 Git Commit을 권장하며, 실패 시 `git reset --hard`를 통해 v1.6.0 Baseline으로 즉시 복귀합니다.
