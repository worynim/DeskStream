# Release Notes - Hangeul Clock

## [v1.8.0] - 2026-04-12
### 🎨 렌더링 엔진 계층 분리 및 자원 최적화 (Canvas Refactoring & Font Optimization)
- **렌더링 엔진(Renderer) 클래스 독립**: 
    - `DisplayManager`에 비대하게 집중되어 있던 비트맵 캐시 관리 및 고수준 그리기 로직(Dither, Zoom, Scale 등)을 별도의 `Renderer` 클래스로 완벽히 격리했습니다.
    - 이제 하드웨어 제어 논리와 시각적 프레임 생성 논리가 분리되어 아키텍처의 유연성과 유지보수성이 극대화되었습니다.
- **Flash 저장 공간 최적화 (폰트 경량화)**: 
    - 프로그램 저장 공간(Flash) 점유율을 낮추기 위해 사용 빈도가 낮은 한글 폰트를 제외한 경량 버전(`u8g2_font_unifont_t_korean1`)으로 교체했습니다.
    - 이 변경을 통해 **약 100KB 이상의 플래시 가용 공간을 확보**하여 시스템 안정성을 제고했습니다.
- **코드 청결도 및 정합성 강화**: 
    - 레거시 코드 파편을 제거하고 모든 그리기 호출을 새로운 렌더러 인터페이스로 단일화했습니다.

## [v1.7.0] - 2026-04-12
### 🏗️ 차세대 아키텍처 통합 및 데이터 안정성 강화 (HAL & Config Management)
- **하드웨어 추상화 레이어 (I2C_Platform) 도입**: 
    - OLED 전송 로직을 별도의 태스크로 분리하여 메인 루프의 병목 현상을 제거했습니다.
    - I2C 버스 제어를 추상화하여 하드웨어 의존성을 낮추고 확장성을 확보했습니다.
- **중앙 집중식 설정 관리 (ConfigManager)**:
    - 모든 시스템 설정을 한 곳에서 관리하며, FreeRTOS 타이머를 이용한 **3초 지연 저장(Lazy Save)** 기능을 구현하여 Flash 메모리 수명을 보존합니다.
    - 설정 변경 시 즉시 메모리에 반영하고, 일정 시간 조작이 없을 때만 물리 메모리에 쓰는 지능형 세이브 엔진을 탑재했습니다.
- **렌더링 신뢰성 및 반응성 개선**:
    - 버튼 조작 시 화면이 일시적으로 사라지던 현상을 해결하고, 1초 대기 없이 즉시 화면이 갱신되는 `setForceUpdate` 트리거를 적용했습니다.
    - I2C 버스 경합(Conflict)을 방지하기 위한 비동기 전송 동기화 로직을 강화했습니다.
- **코드 무결성 확보**: 표준 라이브러리(STL) 헤더 정비 및 클래스 멤버 스코프 전수 조사를 통해 컴파일 안정성을 극대화했습니다.

## [v1.6.0] - 2026-04-12
### 🛡️ 시스템 안정성 고도화 및 부팅 UX 혁신 (System Stabilization & UX)
- **하드웨어 자가 치유 엔진 (I2C Recovery)**: 실시간 I2C 전송 에러 모니터링 시스템을 구축하여, 통신 장애 발생 시 자동으로 버스를 재초기화하고 디스플레이를 복구하는 `recoverI2CBus` 로직을 탑재했습니다.
- **프리미엄 부팅 UX**: 
    - **시작 멜로디**: 전원 인가 직후 경쾌한 상승음(Do-Mi-Sol-Do)을 재생하여 기기 상태를 청각적으로 피드백합니다.
    - **OLED 다중 라인 로그**: 부팅 과정을 1번 화면에 실시간 로그로 출력하며, 비트맵 로딩 및 네트워크 연결 시 진행 상태를 점(`.`) 애니메이션으로 시각화했습니다.
- **원자적 비트맵 캐시 관리 (Atomic Cache)**: LittleFS에서 데이터를 읽을 때 임시 버퍼를 사용하는 트랜잭션 방식을 채택하여, 로딩 중 오류가 발생해도 기존 캐시 데이터가 유실되지 않도록 보호합니다.
- **레이스 컨디션 방어 (Concurrency Safety)**: FreeRTOS 태스크 간 동기화에 타임아웃 가드를 적용하여, 하드웨어 지연 상황에서도 메모리 오염이 발생하지 않도록 설계를 강화했습니다.
- **코드 모듈화 (Modularization)**: 한글 변환 로직을 `HangeulTimeConverter` 클래스로 완전 분리하여 전체 프로젝트의 아키텍처를 표준화했습니다.

## [v1.5.0] - 2026-04-12
### 🏗️ 대규모 아키텍처 리팩토링 및 최적화 (Architectural Refinement)
- **모듈형 구조 전환**: `DisplayManager`와 `WebManager`를 각각 별도의 `.h` 및 `.cpp` 파일로 분리하여 모듈화하고 유지보수성을 극대화했습니다.
- **WebManager 클래스화**: 단순 함수 기반 서버 로직을 클래스로 정형화하여 기능을 캡슐화했습니다.
- **Flash 메모리 최적화 (RAM 절약)**: 거대한 웹 UI HTML 문자열을 `web_pages.h`로 격리하고 `PROGMEM`을 적용하여 RAM 점유율을 대폭 낮췄습니다.
- **동적 비트맵 캐시 엔진**: 폰트 크기에 따라 필요한 만큼만 메모리를 할당하는 `std::vector` 기반 동적 캐시 시스템을 구현하여, 일반 폰트 사용 시 **기존 대비 RAM 사용량을 50% 이상 절감**했습니다.
- **안정성 강화**: LittleFS 마운트 체크 및 메모리 할당 예외 처리(try-catch)를 추가하여 런타임 안정성을 확보했습니다.
- **전역 상수 표준화**: 코드 내 산재해 있던 I2C 주소, 포트, 타이밍 등 모든 매직 넘버를 `config.h`로 통합 관리하도록 개선했습니다.

- **64px 광폭 비트맵 표준화**: 글자당 비트맵 폭을 기존 32px에서 **64px(512바이트)**로 2배 확장하여, 큰 폰트 사용 시 양 옆이 잘리는 문제를 근본적으로 해결했습니다.
- **자연스러운 글자 겹침(Overlap) 구현**: 32px의 글자 간격은 유지하되, 렌더링 시 ±16px 오프셋을 허용하여 넓은 글자가 옆 칸을 자연스럽게 침범하는 고급 렌더링 효과를 적용했습니다.
- **메모리 캐시 최적화**: 64px 규격에 맞춰 `CachedChar` 구조체를 업데이트하고 LittleFS 파일 읽기 로직을 512바이트 단위로 최적화했습니다.

## [v1.3.52] - 2026-04-11 04:20
### 🚀 기초 시스템 최적화 및 안정화 (Stability & Performance)
- **화면 잔상(Ghosting) 완전 해결**: `clearAll()` 메서드 도입으로 모드 전환 시 이전 픽셀 잔상을 물리적으로 소거했습니다.
- **코드 및 상수 표준화**: 코드 내 모든 **`ANIM_` 약어를 `ANIMATION_` 풀네임으로 교체**하고, 모든 모드(Animation, Display, Format)를 0-based 인덱스로 완벽 통일했습니다.
- **폰트 즉시 반영**: 웹에서 폰트를 업로드하는 즉시 **시계의 모든 글자가 새 폰트로 자동 갱신**되는 로직을 구현했습니다.
- **I2C 대역폭 효율화**: `pushParallel()` 전송 로직 단일화 및 중복 전송 명령 제거로 성능을 향상시켰습니다.
- **UI 정돈**: 동작하지 않던 '폰트 초기화' 버튼 삭제 등 웹 인터페이스를 정비했습니다.
